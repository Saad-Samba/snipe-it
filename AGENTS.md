
## Requirements Management
Refer to the following documents when translating stakeholders requirements into Azure DevOps work items.

- ADO_WORKITEM_TYPES.md — definitions and purpose of each ADO work item type  
- ADO_CLASSIFICATION_GUIDE.md — how to classify stakeholder input into the correct work item type  
- ADO_SCHEMAS.md — required fields and JSON schemas for each work item type  
- ADO_LINKING_RULES.md — rules for linking work items using *children* and *related*  

These documents collectively define how work items must be created, structured, and linked.
